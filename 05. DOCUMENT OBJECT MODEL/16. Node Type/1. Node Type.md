# üéØ Node Type

## üìå Penjelasan

Node Type adalah **property** yang digunakan untuk mengetahui **tipe dari sebuah Node** dalam DOM (Document Object Model). Ketika kita menyeleksi Node menjadi NodeList, terkadang kita perlu mengetahui tipe Node tersebut:

- Apakah itu **Element** (seperti `<div>`, `<p>`)
- Apakah itu **TextElement** (teks di dalam elemen)
- Atau tipe lainnya

Node sendiri memiliki property bernama `nodeType`, dimana kita bisa melihat tipe dari node tersebut dalam bentuk **angka konstanta**.

> üìñ **Referensi:** [MDN Web Docs - Node.nodeType](https://developer.mozilla.org/en-US/docs/Web/API/Node/nodeType)

---

## üìä Tabel Node Type

| Constant | Value | Description |
|----------|-------|-------------|
| `Node.ELEMENT_NODE` | **1** | An `Element` node like `<p>` or `<div>`. |
| `Node.ATTRIBUTE_NODE` | **2** | An `Attribute` of an `Element`. |
| `Node.TEXT_NODE` | **3** | The actual `Text` inside an `Element` or `Attr`. |
| `Node.CDATA_SECTION_NODE` | **4** | A `CDATASection`, such as `<!CDATA[[ ‚Ä¶ ]]>`. |
| `Node.PROCESSING_INSTRUCTION_NODE` | **7** | A `ProcessingInstruction` of an XML document, such as `<?xml-stylesheet ‚Ä¶ ?>`. |
| `Node.COMMENT_NODE` | **8** | A `Comment` node, such as `<!-- ‚Ä¶ -->`. |
| `Node.DOCUMENT_NODE` | **9** | A `Document` node. |
| `Node.DOCUMENT_TYPE_NODE` | **10** | A `DocumentType` node, such as `<!DOCTYPE html>`. |
| `Node.DOCUMENT_FRAGMENT_NODE` | **11** | A `DocumentFragment` node. |

## üí° Penjelasan Setiap Node Type

### 1Ô∏è‚É£ `Node.ELEMENT_NODE` (Value: 1)

**Penjelasan:**  
Merepresentasikan **elemen HTML** seperti `<div>`, `<p>`, `<span>`, `<h1>`, dll. Ini adalah tipe node yang paling sering kita gunakan.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <div id="myDiv">Hello World</div>
    
    <script>
        const divElement = document.getElementById('myDiv');
        console.log(divElement.nodeType); // Output: 1
        console.log(divElement.nodeName); // Output: DIV
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- Kita mengambil elemen `<div>` dengan id `myDiv`
- `nodeType` mengembalikan nilai **1** yang berarti ELEMENT_NODE
- `nodeName` mengembalikan nama elemen yaitu **DIV**

**Output:**
```
1
DIV
```

---

### 2Ô∏è‚É£ `Node.ATTRIBUTE_NODE` (Value: 2)

**Penjelasan:**  
Merepresentasikan **atribut** dari sebuah elemen HTML, seperti `class`, `id`, `src`, dll.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <div id="myDiv" class="container">Content</div>
    
    <script>
        const divElement = document.getElementById('myDiv');
        const idAttr = divElement.getAttributeNode('id');
        
        console.log(idAttr.nodeType); // Output: 2
        console.log(idAttr.nodeName); // Output: id
        console.log(idAttr.nodeValue); // Output: myDiv
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- Kita mengambil node atribut `id` menggunakan `getAttributeNode()`
- `nodeType` mengembalikan nilai **2** yang berarti ATTRIBUTE_NODE
- `nodeValue` mengembalikan nilai dari atribut tersebut

**Output:**
```
2
id
myDiv
```

### 3Ô∏è‚É£ `Node.TEXT_NODE` (Value: 3)

**Penjelasan:**  
Merepresentasikan **teks** yang ada di dalam sebuah elemen atau atribut. Teks ini bisa berupa konten yang terlihat di halaman web.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <p id="myParagraph">Ini adalah teks di dalam paragraf</p>
    
    <script>
        const paragraph = document.getElementById('myParagraph');
        const textNode = paragraph.firstChild;
        
        console.log(textNode.nodeType); // Output: 3
        console.log(textNode.nodeName); // Output: #text
        console.log(textNode.nodeValue); // Output: Ini adalah teks di dalam paragraf
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- `firstChild` mengambil child pertama dari paragraf, yaitu text node
- `nodeType` mengembalikan nilai **3** yang berarti TEXT_NODE
- `nodeName` untuk text node selalu **#text**

**Output:**
```
3
#text
Ini adalah teks di dalam paragraf
```

### 4Ô∏è‚É£ `Node.CDATA_SECTION_NODE` (Value: 4)

**Penjelasan:**  
Merepresentasikan **CDATA Section** dalam dokumen XML. CDATA digunakan untuk menyimpan data yang tidak akan di-parse oleh XML parser.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <script>
        const xmlString = `
            <root>
                <![CDATA[Ini adalah CDATA content <tag> tidak akan di-parse]]>
            </root>
        `;
        
        const parser = new DOMParser();
        const xmlDoc = parser.parseFromString(xmlString, "text/xml");
        const cdataNode = xmlDoc.documentElement.firstChild;
        
        console.log(cdataNode.nodeType); // Output: 4
        console.log(cdataNode.nodeName); // Output: #cdata-section
        console.log(cdataNode.nodeValue); // Output: Ini adalah CDATA content <tag> tidak akan di-parse
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- Membuat dokumen XML dengan CDATA section
- Parse XML string menggunakan `DOMParser`
- `nodeType` mengembalikan nilai **4** yang berarti CDATA_SECTION_NODE

**Output:**
```
4
#cdata-section
Ini adalah CDATA content <tag> tidak akan di-parse
```

### 5Ô∏è‚É£ `Node.PROCESSING_INSTRUCTION_NODE` (Value: 7)

**Penjelasan:**  
Merepresentasikan **Processing Instruction** dalam dokumen XML, seperti `<?xml-stylesheet ?>` yang memberikan instruksi kepada aplikasi.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <script>
        const xmlString = `
            <?xml version="1.0"?>
            <?xml-stylesheet type="text/css" href="style.css"?>
            <root>Content</root>
        `;
        
        const parser = new DOMParser();
        const xmlDoc = parser.parseFromString(xmlString, "text/xml");
        const piNode = xmlDoc.childNodes[1]; // Processing instruction node
        
        console.log(piNode.nodeType); // Output: 7
        console.log(piNode.nodeName); // Output: xml-stylesheet
        console.log(piNode.nodeValue); // Output: type="text/css" href="style.css"
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- Membuat XML dengan processing instruction `<?xml-stylesheet ?>`
- Mengakses processing instruction node
- `nodeType` mengembalikan nilai **7** yang berarti PROCESSING_INSTRUCTION_NODE

**Output:**
```
7
xml-stylesheet
type="text/css" href="style.css"
```

### 9Ô∏è‚É£ `Node.DOCUMENT_FRAGMENT_NODE` (Value: 11)

**Penjelasan:**  
Merepresentasikan **fragment dokumen** yang ringan dan minimal. Document Fragment sering digunakan untuk manipulasi DOM yang lebih efisien karena tidak memicu reflow.

**Contoh Kode:**

```html
<!DOCTYPE html>
<html>
<body>
    <ul id="myList"></ul>
    
    <script>
        const fragment = document.createDocumentFragment();
        
        console.log(fragment.nodeType); // Output: 11
        console.log(fragment.nodeName); // Output: #document-fragment
        
        // Membuat beberapa elemen list
        for (let i = 1; i <= 3; i++) {
            const li = document.createElement('li');
            li.textContent = `Item ${i}`;
            fragment.appendChild(li);
        }
        
        // Menambahkan fragment ke DOM (lebih efisien)
        document.getElementById('myList').appendChild(fragment);
    </script>
</body>
</html>
```

**Penjelasan Kode:**
- `createDocumentFragment()` membuat document fragment baru
- `nodeType` mengembalikan nilai **11** yang berarti DOCUMENT_FRAGMENT_NODE
- Fragment digunakan untuk menambahkan multiple elements sekaligus dengan efisien

**Output:**
```
11
#document-fragment
```

**Hasil di Browser:**
- Item 1
- Item 2
- Item 3
