- ğŸ“– **GET** = Membaca menu (mengambil data)

## âš ï¸ Error Handling

Penanganan error yang baik sangat penting untuk aplikasi yang robust.

### Contoh Error Handling Lengkap:

```javascript
async function fetchWithErrorHandling(url) {
    try {
        const response = await fetch(url);
        
        // Cek apakah response berhasil
        if (!response.ok) {
            throw new Error(`HTTP Error! Status: ${response.status}`);
        }
        
        const data = await response.json();
        console.log('Data berhasil diambil:', data);
        return data;
        
    } catch (error) {
        // Cek jenis error
        if (error.name === 'TypeError') {
            console.error('Network error atau URL salah:', error.message);
        } else {
            console.error('Error lainnya:', error.message);
        }
        
        // Return null atau value default
        return null;
    }
}

// Contoh penggunaan
fetchWithErrorHandling('https://jsonplaceholder.typicode.com/posts/1');
```

### Status Code yang Umum:

| Status Code | Arti | Penjelasan |
|-------------|------|------------|
| 200 | OK | Request berhasil |
| 201 | Created | Data berhasil dibuat |
| 400 | Bad Request | Request tidak valid |
| 404 | Not Found | Data tidak ditemukan |
| 500 | Server Error | Error di server |

### Contoh Menangani Status Code:

```javascript
async function fetchWithStatusCheck(url) {
    try {
        const response = await fetch(url);
        
        switch(response.status) {
            case 200:
                const data = await response.json();
                console.log('Sukses:', data);
                break;
            case 404:
                console.error('Data tidak ditemukan!');
                break;
            case 500:
                console.error('Server error!');
                break;
            default:
                console.error(`Status: ${response.status}`);
        }
        
    } catch (error) {
        console.error('Network error:', error);
    }
}

fetchWithStatusCheck('https://jsonplaceholder.typicode.com/posts/999');
```

## ğŸ’¡ Tips & Best Practices

### 1. Gunakan async/await (Bukan Promise Chain)

âœ… **Good:**
```javascript
async function getData() {
    const response = await fetch(url);
    const data = await response.json();
    return data;
}
```

âŒ **Avoid:**
```javascript
function getData() {
    fetch(url)
        .then(response => response.json())
        .then(data => {
            // nested promises...
        });
}
```

### 2. Selalu Tangani Error

```javascript
async function safeRequest() {
    try {
        const response = await fetch(url);
        if (!response.ok) throw new Error('Request failed');
        return await response.json();
    } catch (error) {
        console.error(error);
        return null;
    }
}
```

### 3. Gunakan Destructuring untuk Response

```javascript
async function getUser(id) {
    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
        const { name, email, phone } = await response.json();
        console.log(`Name: ${name}, Email: ${email}`);
    } catch (error) {
        console.error(error);
    }
}
```

### 4. Buat Function Reusable

```javascript
// Function generic untuk semua request
async function apiRequest(endpoint, method = 'GET', body = null) {
    const options = {
        method,
        headers: { 'Content-Type': 'application/json' }
    };
    
    if (body) {
        options.body = JSON.stringify(body);
    }
    
    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com${endpoint}`, options);
        return await response.json();
    } catch (error) {
        console.error('API Error:', error);
        return null;
    }
}

// Penggunaan
apiRequest('/posts/1'); // GET
apiRequest('/posts', 'POST', { title: 'New Post', body: 'Content' }); // POST
```

### 5. Tampilkan Loading State

```javascript
async function fetchWithLoading(url) {
    console.log('Loading...');
    
    try {
        const response = await fetch(url);
        const data = await response.json();
        console.log('Done!', data);
    } catch (error) {
        console.error('Error!', error);
    }
}
```

## ğŸ› Troubleshooting

### Problem 1: CORS Error

**Error:**
```
Access to fetch at '...' from origin '...' has been blocked by CORS policy
```

**Solusi:**
- JSONPlaceholder sudah support CORS
- Jika tetap error, gunakan extension seperti "CORS Unblock" untuk testing
- Di production, backend harus setup CORS dengan benar

### Problem 2: Undefined Response

**Error:**
```javascript
console.log(data.title); // undefined
```

**Solusi:**
```javascript
// Cek dulu apakah data ada
if (data && data.title) {
    console.log(data.title);
} else {
    console.log('Title tidak tersedia');
}
```

### Problem 3: JSON Parse Error

**Error:**
```
SyntaxError: Unexpected token < in JSON
```

**Solusi:**
```javascript
try {
    const data = await response.json();
} catch (error) {
    console.error('Response bukan JSON:', await response.text());
}
```

### Problem 4: Network Error

**Error:**
```
TypeError: Failed to fetch
```

**Solusi:**
- Cek koneksi internet
- Pastikan URL benar
- Cek apakah website tidak diblokir

### Problem 5: Data Tidak Tersimpan

**Masalah:** Data POST/PUT/DELETE tidak permanen

**Penjelasan:**
> â„¹ï¸ JSONPlaceholder hanya fake API untuk testing. Data tidak benar-benar tersimpan. Response hanya simulasi.
